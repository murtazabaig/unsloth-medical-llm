🔧 TECHNICAL SPECIFICATIONS
═══════════════════════════════════════════════════════════════════════════════

Base Model:
  • Model: Llama 2 7B (or Llama 3)
  • Quantization: 4-bit (bitsandbytes)
  • Max Sequence Length: 2048 tokens
  • Framework: PyTorch + Transformers

LoRA Configuration:
  • Rank: 64
  • Alpha: 16
  • Dropout: 0.05
  • Target Modules: Q/K/V/O projections + FFN layers
  • Bias: No

Training Setup:
  • Optimizer: paged_adamw_32bit
  • Learning Rate: 5e-4
  • Batch Size: 4 (effective: 8 with accumulation)
  • Epochs: 3
  • Warmup Steps: 100
  • Weight Decay: 0.01

Data:
  • Medical Q&A Pairs: 20 (sample), expandable
  • Train/Eval Split: 80/20
  • Format: CSV with question/answer columns
  • Domain: Clinical knowledge

🐛 TROUBLESHOOTING QUICK REFERENCE
═══════════════════════════════════════════════════════════════════════════════

Out of Memory (OOM):
  ✗ Fix: Reduce batch_size to 2 or 1
  ✗ Fix: Lower max_seq_length to 1024
  ✗ Fix: Reduce lora_rank to 32

GPU Not Found:
  ✓ Colab: Runtime > Change runtime type > GPU
  ✓ Local: pip install torch --index-url https://download.pytorch.org/whl/cu118

Slow Training:
  ✓ Verify GPU is used: nvidia-smi in terminal
  ✓ Increase batch size (if memory allows)
  ✓ Check Unsloth is properly installed

Poor Medical Responses:
  ✓ Increase training data
  ✓ Train for more epochs
  ✓ Use higher quality medical sources

See README.md for detailed troubleshooting guide.
